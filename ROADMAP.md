# ROADMAP — Smile AI Studio 今後1年の計画

**期間**: 2026年7月 〜 2027年6月（想定）  
**目的**: AI 開発本部としての基盤を固め、全プロジェクトの開発速度と品質を安定させる

---

## 四半期ごとの目標

### Q1（2026年7月〜9月）— 基盤づくり

**テーマ: 「ルールと型を揃える」**

| 目標 | 内容 | 完了の目安 |
|------|------|-----------|
| ドキュメント整備 | README / VISION / RULES / テンプレートの運用開始 | 全員が入口を知っている |
| 3プロジェクトのメタ情報統一 | 各 `projects/*/README.md` にリポジトリ・デプロイ・担当を記載 | 依頼時に迷わない |
| 標準フロー確立 | 依頼 → 実装 → テスト → Push → Netlify → 通知 | 1サイクル以上成功 |
| 第1号 AI エージェント | 汎用開発エージェントを `agents/` に定義 | DEVELOPMENT_RULES を参照して動作 |

**マイルストーン**
- [ ] Smile AI Studio リポジトリの運用開始
- [ ] FuwafuwaPanicManager / GiftCanvas / イベント相談 LP の変更を1件ずつ新フローで実施
- [ ] スマホ依頼テンプレートの社内共有

---

### Q2（2026年10月〜12月）— 効率化と品質

**テーマ: 「繰り返しを自動化する」**

| 目標 | 内容 | 完了の目安 |
|------|------|-----------|
| テスト標準化 | プロジェクトごとの最低限テスト手順を文書化 | 変更ごとに実行可能 |
| Netlify 確認チェックリスト | プロジェクト別の確認項目を `docs/guides/` に整備 | 確認漏れゼロを目指す |
| デプロイ・通知の半自動化 | `scripts/` でログ取得や定型通知の補助 | 手作業時間を短縮 |
| 専用エージェント検討 | LP 用・管理画面用など役割分担 | 1〜2 エージェント追加 |

**マイルストーン**
- [ ] 四半期あたりの開発依頼を新フローで **80%以上** 処理
- [ ] 本番障害・表示崩れのインシデント記録と振り返りテンプレート
- [ ] GiftCanvas とイベント LP の横断改善（表示速度・スマホ UX）1件

---

### Q3（2027年1月〜3月）— 拡張準備

**テーマ: 「次のプロジェクトを迎え入れる」**

| 目標 | 内容 | 完了の目安 |
|------|------|-----------|
| 新規プロジェクト追加手順の実証 | PROJECTS.md の手順で1件追加（または予約管理など候補を具体化） | テンプレートから立ち上げ可能 |
| オンボーディング | `docs/onboarding/` に新メンバー・新エージェント向けガイド | 初日から依頼〜完了ができる |
| セキュリティ・個人情報ガイド | フォーム・管理画面向けの最低限ルール | 依頼時チェック項目化 |
| KPI の見える化 | 依頼件数・リードタイム・完了率の簡易集計 | 月1回レビュー |

**マイルストーン**
- [ ] 新規プロジェクト（または主要機能）1件を Smile AI Studio 管轄に追加
- [ ] エージェント設定が `agents/configs/` でバージョン管理されている
- [ ] 1年計画の中間レビュー実施

---

### Q4（2027年4月〜6月）— 成熟と次の一年

**テーマ: 「文化として定着させる」**

| 目標 | 内容 | 完了の目安 |
|------|------|-----------|
| 開発文化の定着 | VISION に沿った振り返り・事例集 | チーム外にも説明できる |
| プロジェクト横断の共通コンポーネント検討 | ボタン・フォーム・通知など共通化の可否調査 | 無理に共通化しない判断も記録 |
| 次年度ロードマップ草案 | 2027〜2028 の優先事業と開発投資 | 経営・現場で合意 |
| AI エージェント群の整理 | 役割・権限・エスカレーションルールの明文化 | エージェント追加がルーティン化 |

**マイルストーン**
- [ ] 1年間の開発完了通知・インシデントの振り返りレポート
- [ ] ROADMAP 2027-2028 草案公開
- [ ] 「スマホから依頼して安心して届く」状態の定着確認

## リスクと対策

| リスク | 対策 |
|--------|------|
| 依頼が口頭のまま残る | REQUEST_TEMPLATE の徹底・短文化 |
| AI が既存機能を壊す | DEVELOPMENT_RULES・最小差分・確認フロー |
| プロジェクトごとにルールがぶれる | Smile AI Studio を唯一の入口にする |
| エージェントが増えて管理不能 | `agents/README.md` で役割一覧・設定のバージョン管理 |
| Netlify 確認の省略 | 完了通知に URL と確認結果を必須項目化 |
