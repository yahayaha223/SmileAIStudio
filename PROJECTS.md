## プロジェクト横断の共通事項

すべてのプロジェクトで以下を共通ルールとします。

- [DEVELOPMENT_RULES.md](./DEVELOPMENT_RULES.md) に従う
- 変更前に対象プロジェクト名を明記する
- GitHub へ Push 後、Netlify で表示確認する
- 完了時は依頼者へ通知する

## 今後追加予定のプロジェクト

新規プロジェクトは、事業拡大・ツール化ニーズに応じて順次 Smile AI Studio 管轄に追加します。

| 区分 | 候補例 | 追加の目安 |
|------|--------|-----------|
| **イベント系** | 予約管理、参加者向けマイページ、当日用ダッシュボード | イベント事業のデジタル化が進むタイミング |
| **ギフト・EC系** | 注文管理、配送連携、ギフトカード機能 | GiftCanvas との連携強化時 |
| **社内効率化** | 請求・在庫・スタッフシフト連携ツール | 運用負荷が定型化されたタイミング |
| **マーケ・LP系** | キャンペーン LP、事例ページ、メディア向けページ | 施策ごとの短期 LP 需要 |
| **データ・分析** | コンバージョン集計、イベント KPI ダッシュボード | 計測基盤が整った段階 |

### 新規プロジェクト追加時の手順

1. `projects/<プロジェクト名>/` ディレクトリを作成する
2. `README.md` に概要・リポジトリ・デプロイ先・担当を記載する
3. 本ファイル（PROJECTS.md）の「現在管理中」に追記する
4. 必要に応じて `agents/` に専用エージェント設定を追加する
5. [ROADMAP.md](./ROADMAP.md) を更新する
